# 智能体工程协作指南

本文件用于约束智能体、自动化编码助手和后续维护者在本仓库中的开发、文档、数据、安全和验证行为。处理任何任务时，应优先遵守本文件中的项目边界与操作规则。

## 一、项目上下文

### 项目名称

- AI跨境选品分析平台
- cross-border-ai-selection-platform

### 项目定位

本仓库是一个面向跨境轻小件选品场景的 AI 决策辅助平台，围绕商品数据分析、利润评估、竞争强度判断、物流与风险识别、候选池管理、可视化看板和 AI 选品建议构建。

当前样例数据以手机支架相关轻小件商品作为种子品类，用于验证选品分析流程。不要把项目描述为长期只服务手机支架品类。后续迭代方向包括接入 1688 API、Amazon API、扩展更多轻小件品类、扩大候选商品规模，并提升市场与供应链数据时效性。

本仓库公开内容应定位为脱敏后的产品化演示版本，项目数据、截图、文档和示例配置不得包含真实业务敏感信息、真实密钥、真实客户数据或内部未公开资料。

## 二、核心协作原则

1. 只处理用户当前明确要求的任务。
2. 保持改动范围收敛，并与现有架构和代码风格一致。
3. 不为了小任务做大范围重构。
4. 除非任务明确要求，否则优先保持现有接口、数据结构和用户可见行为稳定。
5. 文档类任务不得修改业务逻辑、运行行为、依赖版本或数据库结构。
6. 如果确实需要改动业务逻辑，必须说明原因、影响范围和验证方式。
7. 品类扩展应优先采用数据驱动或配置化方式，不应把当前种子品类写死为长期业务边界。
8. 尊重工作区中已经存在的用户改动，不回滚无关变更。
9. 如果用户明确要求功能升级、依赖引入或工程化重构，可以在说明影响范围和验证方式后执行。
10. 每次改动完成后，应简要说明本次改动内容、涉及文件和建议验证方式。

## 九、前端工程约定

前端采用 React + Vite 构建，页面组件应优先关注状态管理和 UI 展示，不应直接拼接底层请求逻辑。

请求层统一放在：

```
client/src/services/
```

推荐结构：

```
client/src/services/
├── http.js          # Axios 实例、baseURL、timeout、响应拦截器、错误归一化
├── productApi.js    # 商品列表、商品详情、查询参数组装
├── dashboardApi.js  # Dashboard 数据读取
├── favoriteApi.js   # 候选池读取、新增、删除和缓存更新
├── aiApi.js         # AI 对话、AI 商品报告、消息裁剪和错误兜底
├── index.js         # 统一导出业务 API
└── api.js           # 兼容旧导入路径的 re-export
```

前端改动应遵守：

1. 新业务接口优先通过 `client/src/services/` 封装。
2. 页面组件不直接硬编码后端 URL。
3. 候选池和 AI 相关请求应自动携带匿名 `x-client-id`。
4. 保持 Loading / Error / Empty / 同步中状态一致。
5. 涉及缓存策略时，应确保缓存数据和后台刷新数据不会造成页面状态错乱。
6. 改动旧导入路径时，应优先保留 `api.js` 兼容导出，避免遗漏 import 导致页面运行失败。

## 十、后端工程约定

后端采用 Node.js + Express，并通过 Vercel Serverless Functions 方式部署。

后端改动应遵守：

1. API 路径统一以 `/api` 为前缀。
2. 商品主数据从 Supabase `products` 表读取。
3. 候选池数据写入 Supabase `favorites` 表。
4. 匿名访问者通过 `x-client-id` 区分候选池数据。
5. 商品利润率、竞争等级、风险标签、推荐评分等字段应由统一工具函数补充，避免在多个路由中重复实现。
6. AI Provider 调用应封装在 service 层，不应直接写在路由业务中。
7. AI 相关异常应映射为可读错误信息，并提供规则化兜底结果，避免前端流程中断。
8. 新增环境变量时，必须同步更新 `.env.example` 和 README 环境变量说明。

## 十一、产品迭代边界

当前种子品类：

- 手机支架相关轻小件商品

当前平台方向：

- 轻小件跨境选品分析
- 商品指标分析
- 候选商品沉淀
- AI 辅助决策
- 数据看板与业务验证

预期迭代方向：

- 接入 1688 API，提升货源、采购成本、供应商、起订量和发货地等数据时效性。
- 接入 Amazon API，提升候选商品、销量、评论、评分和竞争信号更新能力。
- 从当前种子品类扩展到更多跨境轻小件商品。
- 增加登录认证、用户体系和权限管理，支持不同用户沉淀独立候选池与分析记录。
- 强化 AI 建议与结构化指标之间的一致性，使回答更稳定地引用利润率、竞争指数、风险标签、物流成本和候选池上下文。
- 完善部署、监控、缓存、异常兜底、数据刷新和增量同步策略。
