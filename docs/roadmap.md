# Roadmap

## 路线图定位

当前版本是AI跨境选品分析平台的产品与工程基线。它已经覆盖商品列表、详情、Dashboard、选品分析、候选池、Supabase 数据读取、AI 对话、AI 商品报告和缓存优化等核心流程。

后续路线重点是在现有流程基础上接入真实数据源、扩大选品规模、提升数据时效性，并逐步增强 AI 辅助决策质量。当前手机支架商品池是种子品类，用于验证轻小件跨境选品分析流程，不是平台长期唯一支持的品类。

## 当前基线

当前版本已形成产品、数据、AI 和工程维护四个方面的基础能力：平台已覆盖手机支架种子品类商品池、Supabase `products` 商品主数据、JSON 备份数据、利润 / 竞争 / 风险 / 推荐评分计算、Dashboard 数据看板、商品列表、详情、选品分析、候选池、匿名 `client_id` 候选池机制、AI 选品对话、AI 商品报告、前端缓存和后端短期缓存等核心流程。

同时，README 已作为项目入口，`docs/` 已沉淀产品概览、技术架构、数据模型、性能优化和产品路线图；公开文档统一使用“AI跨境选品分析平台”和“轻小件跨境选品分析”表述，明确手机支架是当前种子品类而不是长期唯一品类，并将 `.env`、私有草稿、个人记录等内容与公开仓库隔离。

这些能力为后续接入真实 API、扩展更多跨境轻小件品类、落地登录权限体系和扩大选品规模提供了基础结构。

## 中期方向：扩展更多轻小件品类

目标：从单一种子品类扩展为更通用的跨境轻小件候选商品池。

计划能力：

- 补充更多轻小件样例品类，例如桌面收纳、旅行配件、车载小配件、手机周边、厨房小工具、宠物小用品等。
- 将类目和标签从固定样例逐步转为数据驱动。
- 让筛选、排序、Dashboard 和 AI 报告支持跨品类对比。
- 保持现有商品字段和接口结构稳定，避免为了品类扩展破坏已完成流程。

## 中期方向：接入 1688 API

目标：提升货源侧数据覆盖和时效性。

计划能力：

- 拉取或导入 1688 商品基础信息。
- 获取采购价格、起订量、供应商、发货地、图片等字段。
- 将 1688 数据映射到当前商品数据模型。
- 增加数据去重、字段兜底和异常处理。
- 为利润测算提供更接近真实采购侧的数据。

需要注意：

- API 权限、调用频率、字段稳定性和合规限制需要单独评估。
- 真实货源数据可能存在价格区间、规格差异和供应商波动，需要在数据模型中保留不确定性。

## 中期方向：接入 Amazon API

目标：提升市场侧数据覆盖和时效性。

计划能力：

- 获取候选商品的市场表现数据。
- 同步价格、评分、评论数量、排名或其他可用竞争信号。
- 将 Amazon 数据与当前商品池做匹配。
- 更新竞争指数、需求判断和风险判断依据。
- 为 AI 分析提供更实时的市场上下文。

需要注意：

- Amazon API 的权限、费用、频率限制和数据字段需要单独评估。
- 市场数据需要处理刷新频率、历史快照和异常波动。

## 长期方向：扩大选品规模

目标：从固定样本商品池扩展为可持续更新的候选商品池。

计划能力：

- 支持更多跨境轻小件品类。
- 增加候选商品批量导入和批量分析。
- 增加商品分组、标签、优先级和观察状态。
- 对候选池建立更清晰的跟进流程。
- 支持按品类、利润、竞争、风险和供应链信号进行持续观察。

## 长期方向：增强 AI 决策质量

目标：让 AI 建议更稳定地服务选品决策，而不是只做泛化回答。

计划能力：

- 让 AI 明确引用利润、竞争、风险、物流、评分等结构化指标。
- 为 AI 商品报告增加固定分析框架。
- 对 AI 建议进行可解释性和一致性检查。
- 增加候选池对比分析。
- 对高风险商品给出更具体的验证建议。

## 中长期方向：登录认证与权限管理

目标：从匿名 `client_id` 候选池逐步升级到用户级候选池和分析记录。

计划能力：

- 增加登录认证和用户身份体系。
- 将候选池从浏览器匿名隔离升级为用户级持久化。
- 支持用户保存商品分析记录、跟进状态和历史报告。
- 为后续角色权限、团队协作和更精细的数据访问控制预留边界。

## 长期方向：运维与数据质量

目标：让平台更适合长期运行和持续维护。

计划能力：

- 数据刷新任务和刷新记录。
- API 失败重试和降级策略。
- 数据质量校验。
- 缓存命中率和接口耗时监控。
- 部署回归检查。
- 公开 changelog 和版本说明。

## 暂不优先处理的方向

除非出现明确业务需求，以下能力暂不作为优先路线：

- 复杂企业级 RBAC。
- 多租户后台。
- 支付功能。
- 大规模微服务拆分。
- 与轻小件跨境选品分析无关的业务模块。
