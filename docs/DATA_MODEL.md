# Data Model

## 文档定位

本文档说明极瑞AI跨境选品分析平台当前使用的核心数据模型。平台面向跨境轻小件选品场景，当前以手机支架作为种子品类和演示数据，用于验证商品池、候选池、利润评估、风险识别、推荐评分、Dashboard 和 AI 分析流程。

当前数据模型不局限于手机支架，后续可扩展到桌面收纳、旅行配件、车载小配件、手机周边、厨房小工具、宠物小用品等更多跨境轻小件品类。

## 当前数据来源

当前数据层主要由三部分组成：

- Supabase PostgreSQL `products` 表：商品主数据来源。
- Supabase PostgreSQL `favorites` 表：候选池收藏数据来源。
- `server/data/products.json`：商品备份数据和种子数据来源。

后端读取商品数据后，会补充利润、竞争、风险和推荐评分等计算字段，再返回给前端页面和 AI 分析流程使用。

## 商品数据字段

`products` 表和 JSON 备份数据共享同一套核心商品字段。每一条记录表示一个跨境轻小件候选商品；当前示例值多来自手机支架种子品类。

> 说明：`amazonPrice` 使用美元，`cost1688` 和 `shippingCost` 使用人民币。利润测算时需要先统一币种再计算。

| 字段名 | 类型 | 含义 | 示例 | 用途 |
| --- | --- | --- | --- | --- |
| `id` | number | 商品唯一标识 | `1` | 用于商品详情查询、收藏关联、列表渲染 key |
| `productName` | string | 商品名称 | `可折叠桌面手机支架` | 用于列表展示、搜索、详情页标题 |
| `category` | string | 商品类型 | `桌面支架` | 用于类目筛选、Dashboard 类目统计 |
| `amazonPrice` | number | Amazon 预估售价，单位美元 | `19.99` | 用于销售额、平台手续费、利润率计算 |
| `cost1688` | number | 1688 采购成本，单位人民币 | `14.2` | 用于单件成本和利润计算 |
| `shippingCost` | number | 单件预估物流成本，单位人民币 | `6.8` | 用于利润测算、物流成本分析、风险判断 |
| `platformFeeRate` | number | 平台手续费比例 | `0.15` | 用于平台手续费计算 |
| `estimatedMonthlySales` | number | 预估月销量 | `920` | 用于销量分析、推荐评分、Dashboard 统计 |
| `rating` | number | 商品评分 | `4.3` | 用于口碑评估、风险判断、排序 |
| `reviewCount` | number | 评论数 | `2480` | 用于判断市场成熟度、竞争强度和可信度 |
| `competitionScore` | number | 竞争指数，范围 0-100 | `82` | 用于竞争等级判断、排序、风险分析 |
| `weight` | number | 商品重量，单位 kg | `0.31` | 用于物流成本分析、轻小件判断 |
| `volumeLevel` | string | 体积等级，只用 `small`、`medium`、`large` | `small` | 用于物流难度分析、风险等级判断 |
| `material` | string | 主材质 | `ABS+TPU` | 用于详情展示、风险说明、卖点分析 |
| `supplier` | string | 模拟 1688 供应商名称 | `深圳市驰稳车品厂` | 用于货源展示，后续也可扩展供应商筛选 |
| `image` | string | 商品图片路径 | `/images/products/desktop-metal-stand-01.png` | 用于列表卡片和详情页图片展示 |
| `tags` | string[] | 商品标签数组 | `["桌面", "热销", "轻小件"]` | 用于搜索、筛选、标签展示 |
| `riskFactors` | string[] | 初始风险因素数组 | `["同质化较多"]` | 用于风险等级展示和分析页说明 |
| `recommendationReason` | string | 推荐理由 | `轻小件属性明显，适合观察利润与竞争表现` | 用于详情页和分析页解释推荐原因 |

## 后端计算字段

后端会基于商品原始字段补充计算字段，供前端展示、排序、Dashboard 统计和 AI 上下文使用。

常见计算字段包括：

- `revenueCNY`：预估人民币销售收入。
- `platformFee`：平台手续费。
- `totalCost`：总成本。
- `profit`：单件利润。
- `profitRate`：利润率小数值。
- `profitRatePercent`：利润率百分比。
- `competitionLevel`：竞争等级。
- `riskLevel`：风险等级。
- `riskFactors`：风险原因数组。
- `recommendationScore`：推荐评分。
- `recommendationLevel`：推荐等级。

这些字段用于识别高潜力商品、高风险商品、低竞争高利润商品，以及生成 AI 选品建议。

## 候选池数据

候选池由 Supabase `favorites` 表保存，用于记录某个匿名访问者关注的商品。

核心关系：

| 字段 | 含义 |
| --- | --- |
| `client_id` | 前端生成的匿名访问者标识 |
| `product_id` | 被收藏的商品 id |
| `created_at` | 收藏创建时间 |

前端通过请求头 `x-client-id` 传递匿名标识，后端基于该标识读写候选池。这个机制适合轻量候选池管理，不等同于完整登录鉴权系统。

## AI 上下文数据

AI 对话和商品报告会使用后端提供的商品上下文。上下文通常来自：

- 当前商品池。
- 单个商品详情。
- 当前访问者候选池。
- 利润、风险、竞争、评分、销量等计算字段。
- 商品标签、推荐理由和风险因素。

AI 只基于后端提供的数据进行分析，不应编造不存在的商品、供应商或真实平台数据。

## 前端缓存数据

前端会缓存部分接口数据，用于提升二次访问和接口较慢场景下的体验：

- 完整商品池。
- Dashboard 数据。
- 商品详情。
- 候选池数据。

缓存只用于体验优化，不是最终数据源。后端 Supabase 数据和接口返回仍是页面数据更新的主要依据。

## 当前种子品类说明

当前商品样本以手机支架为主，包括桌面支架、车载支架、折叠支架、磁吸支架、直播支架、懒人支架等细分类目。这些数据用于验证跨境轻小件选品平台的核心流程：

- 商品基础信息展示。
- 类目和标签筛选。
- 利润与成本拆解。
- 物流友好度判断。
- 风险因素识别。
- 推荐评分计算。
- 候选池收藏。
- AI 选品建议。

后续扩展更多轻小件品类时，应优先保持字段结构稳定，通过新增商品数据、类目和标签扩展选品范围。

## 后续扩展方向

- 从固定种子品类扩展到更多跨境轻小件品类。
- 将类目、标签和 AI 检索关键词逐步改为数据驱动。
- 接入 1688 API 后补充更实时的采购、供应商和起订量数据。
- 接入 Amazon API 后补充更实时的市场表现、评分、评论和竞争信号。
- 增加数据质量校验，避免字段缺失或单位不一致影响利润和风险判断。
